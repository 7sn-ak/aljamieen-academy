# دليل أكاديمية الجامعيين — التشغيل والنشر

منصّة خاصة لمتابعة المواد العلمية للطلاب (الفئة العمرية ١٢–١٤). هذا الدليل يشرح كل شيء: كيف تستخدمها الآن، وكيف تجعلها موقعاً حقيقياً بحسابات وإشعارات على الجوال.

---

## ١) ما الذي بين يديك الآن

تطبيق ويب كامل (نموذج عامل) في مجلد «أكاديمية الجامعيين». افتح **`index.html`** بالنقر المزدوج ليشتغل في المتصفح مباشرة.

- زر **«عرض: الطالب ⇄»** أعلى اليسار يبدّل بين **تطبيق الطالب** و**لوحة المشرف**.
- كل تغييراتك (إضافة درس، تفعيل مادة، شهادة، قبول طالب…) **تُحفظ تلقائياً** وتبقى بعد إغلاق الصفحة. لإرجاع كل شيء للوضع الأصلي: لوحة المشرف ← «↺ إعادة تعيين بيانات النموذج».

### ميزات الطالب
الدخول الآمن، الدروس مرتّبة بالتاريخ مع فلترة بالمادة، صفحة الدرس (يوتيوب أو قراءة كتاب)، التمارين (تُحلّ مرة واحدة وتُحتسب النقاط)، المتصدّرون (أفضل ٥ بالنقاط أو الالتزام)، الشهادات القابلة للطباعة، والإشعارات.

### ميزات المشرف
إدارة المواد (إظهار/إخفاء)، إضافة شهادة لكل مادة، إدارة الدروس (إضافة/تعديل/حذف/جدولة بتاريخ مستقبلي)، محرّر التمارين، متابعة الطلاب، قبول/رفض/تعطيل/طرد الحسابات، وإرسال إشعارات للجميع أو لطالب محدّد.

---

## ٢) الفرق بين «النموذج» و«الموقع الحقيقي»

النموذج الحالي يعمل على جهاز واحد ببيانات على المتصفح فقط — ممتاز للعرض والتجربة. ليصبح موقعاً حقيقياً يستخدمه ١٠٠ طالب بحساباتهم وإشعاراتهم، نربطه بخدمة **Firebase** المجانية (من Google). الخطوات التالية تشرح ذلك. هذه الخطوة تقنية بعض الشيء — تقدر تسوّيها بنفسك بالخطوات أدناه، أو يساعدك مطوّر في ساعة تقريباً (كل الملفات جاهزة).

---

## ٣) الربط بـ Firebase (خطوة بخطوة)

### أ. إنشاء المشروع
1. ادخل <https://console.firebase.google.com> بحساب Google.
2. **Add project** ← سمِّه `aljamieen-academy` ← أكمِل (تقدر تعطّل Analytics).

### ب. تفعيل الخدمات الثلاث
- **Authentication** ← Get started ← فعّل **Email link (passwordless sign-in)**.
- **Firestore Database** ← Create database ← ابدأ بوضع Production.
- **Cloud Messaging** (مفعّلة تلقائياً — للإشعارات).

### ج. ربط الإعدادات
1. Project settings (⚙️) ← Your apps ← أيقونة الويب `</>` ← سجّل التطبيق.
2. انسخ قيم `firebaseConfig` والصقها في الملف: **`firebase/firebase-config.js`**.
3. لمفتاح الإشعارات: Project settings ← Cloud Messaging ← Web Push certificates ← **Generate key pair** ← انسخه في `VAPID_KEY` بنفس الملف.
4. اكتب بريدك في `FIRST_ADMIN_EMAIL` لتصبح أول مشرف.

### د. قواعد الأمان
افتح Firestore ← تبويب **Rules** ← الصق محتوى **`firebase/firestore.rules`** ← **Publish**.
هذه القواعد تضمن: الطالب يرى بياناته فقط، والمشرف يدير كل شيء، ولا يحلّ الطالب التمرين مرتين.

### هـ. تفعيل كود الربط (مكتوب وجاهز)
كود الربط الكامل مكتوب في **`firebase/firebase-integration.js`** (مصادقة برابط الإيميل + موافقة المشرف، مزامنة لحظية للبيانات، كل عمليات المشرف والطالب، وإشعارات FCM). لتفعيله:
1. عبّئ `firebase/firebase-config.js` و`firebase-messaging-sw.js` بمفاتيحك.
2. في `index.html` أزل التعليق `<!-- -->` عن سطري تحميل `firebase-config.js` و`firebase-integration.js`.
3. انشر على https وجرّب.

> مهم: هذا الكود مكتوب بعناية لكنه لم يُختبر بعد على مشروع Firebase حقيقي (يحتاج حسابك). عند أول تشغيل قد يحتاج ضبطاً بسيطاً — جرّبه معي في جلسة قادمة أو مع مطوّر، وأي خطأ نصلحه بسرعة. وحتى تفعّله، يبقى التطبيق يعمل في وضع العرض المحلي دون أي تعطّل.

## ٤) نشر الموقع (استضافة مجانية)

أسهل خيار **Firebase Hosting**:
1. ثبّت Node.js ثم نفّذ في مجلد المشروع: `npm i -g firebase-tools` ثم `firebase login`.
2. `firebase init hosting` ← اختر المشروع ← مجلد النشر `.` (الحالي) ← Single-page: No.
3. `firebase deploy` ← يعطيك رابطاً مثل `https://aljamieen-academy.web.app`.

بدائل مجانية أخرى بنفس السهولة: **Netlify** أو **Vercel** (تسحب المجلد وتطلع لك رابط).

بعد النشر على رابط https، يصير الموقع **قابلاً للتثبيت كتطبيق** على الجوال (إضافة إلى الشاشة الرئيسية) ويعمل دون اتصال جزئياً.

---

## ٥) كيف تصبح مشرفاً وتقبل الطلاب

- أول دخول ببريدك (`FIRST_ADMIN_EMAIL`) يمنحك صلاحية المشرف.
- الطالب يفتح الموقع ← «طلب انضمام» ← يكتب اسمه وبريده.
- يظهر طلبه في لوحة المشرف ← «الطلاب» ← **قبول** ← يصله **تلقائياً** رابط دخول على بريده (بدون أي إرسال يدوي منك).
- تقدر **تعطيل** حساب مؤقتاً (يوقف دخوله مع حفظ بياناته) أو **طرده** نهائياً. المعطّل/المطرود يُحجب عنه الموقع برسالة «يُرجى مراجعة المشرف».
- الطالب يدخل **مرة واحدة**؛ تبقى جلسته محفوظة على جهازه، وبياناته (تقدّمه ونقاطه وشهاداته) محفوظة في قاعدة البيانات لا تضيع حتى لو سجّل خروج أو غيّر الجهاز.

---

## ٦) الإشعارات على الجوال

عبر **Firebase Cloud Messaging (FCM)**:
- عند أول دخول يُطلب من الطالب السماح بالإشعارات؛ يُحفظ «رمز جهازه».
- المشرف من «إرسال إشعار» يختار: **للجميع** أو **لطالب محدّد** ← يصل كإشعار على الهاتف فوراً.
- تُرسل إشعارات تلقائية أيضاً عند نشر درس جديد.

---

## ٧) نموذج البيانات (Firestore)

| المجموعة | الوصف | أهم الحقول |
|---|---|---|
| `students/{uid}` | حساب كل طالب | name, email, role(student/admin), status(pending/active/disabled), points, progress, joined |
| `subjects/{id}` | المواد | name, active, cert:{enabled,title,signer} |
| `lessons/{id}` | الدروس | title, type(video/book), course, date, youtube/book, pages, desc |
| `exercises/{lessonId}` | تمارين الدرس | [{q, opts[], answer}] |
| `quizResults/{id}` | نتائج الطالب | uid, lessonId, correct, total, points |
| `notifications/{id}` | الإشعارات | title, body, audience(all)/target(uid), time |
| `deviceTokens/{uid}` | رموز الإشعارات | token |

---

## ٨) الخطوات القادمة المقترحة

- **الشارات** (Badges): تحديد قائمتها (مثل: حافظ المتن، ملتزم الأسبوع، متقن التجويد) ومنحها تلقائياً.
- **تقرير ولي الأمر**: حساب لولي الأمر + ملخص أسبوعي تلقائي.
- **تنويع التمارين**: صح/خطأ، توصيل، إكمال فراغ.
- **حضور وواجبات** للمشرف.
